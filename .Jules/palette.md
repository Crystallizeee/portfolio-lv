## 2024-03-21 - Icon-Only Buttons Missing ARIA Labels
**Learning:** The application extensively uses Lucide icons (`<i data-lucide="..."></i>`) for key interactive elements like mobile menus, chat widgets, sidebar toggles, and scroll-to-top buttons. These buttons lack descriptive `aria-label` attributes, making them inaccessible to screen readers as there is no visible text.
**Action:** Always verify icon-only buttons (`<button>` tags wrapping solely `<i data-lucide="..."></i>` or SVG elements) include explicit `aria-label` attributes describing their function, particularly in core layout components like `app.blade.php` and `admin.blade.php`.
## 2026-07-04 - Added Loading State to Livewire Async Submit Button
**Learning:** Adding explicit loading states to asynchronous form submissions (like Livewire's `saveAlias` action) is a recurring UX pattern that prevents double-submissions and provides crucial user feedback during network requests. In this application, modifying buttons to include `wire:loading.attr="disabled"`, `wire:target`, and swapping text with a spinner using `wire:loading` and `wire:loading.remove` is a highly effective way to achieve this.
**Action:** When implementing or fixing async submit buttons in Livewire components, proactively add loading state visual indicators (e.g., Lucide spinner `<i data-lucide="loader-2" class="animate-spin"></i>`) and disable the button while processing.
